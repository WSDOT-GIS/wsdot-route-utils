getRouteParts

  • getRouteParts(routeId: string, throwErrorOnMatchFail?: boolean, canIncludeDirection?: boolean): Array<string | null> | null
    • Defined in wsdot-route-utils.ts:134

    Splits a state route identifer into its component SR, RRT, and RRQ parts. If the input route ID is not in the expected format, one of two things will happen according to the value of the "throwErrorOnMatchFail" parameter. If set to false, null will be returned. If set to true, an Error will be thrown.

    Parameters

    • routeId: string

      A state route identifier.

    • Default value throwErrorOnMatchFail: boolean = false
    • Default value canIncludeDirection: boolean = false

    Returns Array<string | null> | null

    An array of three elements: SR, RRT, and RRQ. The elements at position 1 and 2 may be null if a route has no RRT or RRQ (as would be the case with a mainline). Will be null if the routeId is not in the expected format and if throwErrorOnMatchFail is false.
